# Changelog

Все заметные изменения в этом проекте документируются здесь.
Копия рядом с `AutoRAW.exe` обновляется при каждой сборке через `build-and-pack.ps1`.

---

## [0.5.1] — 2026-05-14

### Добавлено
- **Установщик Windows (Inno Setup 6, русский UI):** `installer\AutoRAW.iss`, `build-installer.ps1`, `bat\Build-Installer-RU.cmd`, пункт **5** в `dev.cmd`. Итоговый файл **`dist\AutoRAW-Setup-ru.exe`**. Перед копированием файлов приложения при отсутствии **.NET 8 Windows Desktop Runtime** выполняется загрузка и тихая установка официального пакета (`PrepareToInstall` в скрипте Inno).
- **Оформление мастера установки:** `WizardImageFile` / `WizardSmallImageFile` — PNG из `assets\images\` (`installer_banner.png`, `installer_small.png`); **`SetupIconFile`** — `assets\images\setup.ico`.
- **Меню «Профиль»:** пункт **«Перейти в мой каталог профилей…»** — открывает `%LocalAppData%\AutoRAW\user files\Profile` в Проводнике.

### Исправлено
- **Inno:** `AppId` задан литералом `{{E7C3A9B1-82F4-4D6A-9C1E-3F5A8B2D6C90}}` вместо `AppId={{#MyAppGuid}}` — иначе ISCC выдавал ошибку «Unknown constant».
- **`build-installer.ps1`:** текст сообщений в консоли переведён на **ASCII**, чтобы запуск из `cmd` без UTF-8 не ломал разбор строк PowerShell 5.1.

### Изменено
- На машине разработчика для сборки установщика по-прежнему нужен **Inno Setup 6** (`ISCC.exe` в PATH, стандартный путь или переменная **`INNO_SETUP_ISCC`**).

## [0.5.0] — 2026-05-13

### Добавлено
- **Версия продукта** `0.5.0 Alpha`: задаётся в `AutoRAW.csproj` (`Version`, `InformationalVersion`), отображается в заголовке окна, на экране загрузки, в «Справка → О программе…».
- **Экран загрузки** при старте: название AutoRAW, индикатор прогресса, полоска статуса в реальном времени, строка версии; минимум **2 секунды** на экране (при более долгой инициализации останется статус и прогресс; для будущего — API `SetBootProgress` / `SetBootProgressIndeterminate` на сплэше).
- **Инструкция и ченджлог** открываются во встроенном просмотрщике: **Markdig** + стили в духе GitHub (таблицы, блоки кода, цитаты, ссылки открываются во внешнем браузере).
- **О программе…** — отдельное окно: команда **WhiteCube** ([белаякоробка.рф](https://белаякоробка.рф/)), разработчик **Радыгин Иван**.
- **Профили в `user files\Profile\`** рядом с exe: для каждого профиля папки `reference`, `zona`, `setting` (копия выбранного `.xmp`) и `profile.json`. Снижает риск «сломать» профиль, если исходные папки на диске потом удалить или переименовать.
- **Черновик «Несохранённые изменения»** в расширенном режиме: при смене путей reference/zona относительно выбранного профиля или при правке цвета автоматически появляется пункт меню и активируется черновик.
- **Кнопки в расширенном режиме**: «Применить к исходному профилю…» (перезапись пакета исходного пользовательского профиля; для «Кроссовок» — подсказка сохранить как новый) и «Сохранить как новый профиль…».
- **Диалог при смене профиля из меню**, если есть несохранённый черновик: сохранить / отменить правки / остаться в черновике.
- **Миграция** старого `%AppData%\AutoRAW\products.json` в новые пакеты при первом запуске (если `user files\Profile` пуст); исходный файл переименовывается в `.migrated.bak`.
- **`dev.bat`** в корне — вызывает `dev.cmd`.
- **Превью**: при пересчёте (смена папок, профиля, строки таблицы, ползунка анализа, переключение простой/расширенный режим) отображаются **текст статуса** и **индикатор загрузки** над превью, чтобы не казалось зависанием.
- **Простой режим — ползунок «Анализ (длинная сторона)»** в группе «Превью» (тот же параметр, что в расширенном режиме); при изменении превью пересчитывается с индикатором.
- **Экспорт WebP или JPEG**: галочка под кнопкой запуска (простой режим) и в шапке расширенного режима; по умолчанию **WebP**, без галочки — **JPEG**. Если папка выхода **не задана**, файлы пишутся в `<папка товара>\webp\` или `\jpg\` с именами `<stem>.webp` / `<stem>.jpg` (без суффикса `_crop`). Настройка сохраняется в `%AppData%\AutoRAW\export_prefs.json`.

### Изменено
- **Интерфейс**: более компактная вёрстка простого и расширенного режима; кнопка «Запустить кадрирование» выше; светлый «профессиональный» стиль; меню «Вид» — «Показать журнал» (по умолчанию журнал скрыт).
- **Галочка цветокоррекции**: при старте и при смене профиля сбрасывается; при включении — модальное предупреждение про альфа-режим и кнопка «Продолжить».
- **Кнопка «Запустить кадрирование»**: при наведении — зелёный `#39940e`; в отключённом состоянии заметнее (светлый фон, тёмный текст, рамка).
- **Новый профиль** и **сохранение из черновика** пишут данные через `UserProfileBundleService` в `user files\Profile`, а не голые пути в JSON в AppData.
- **`bat\_ensure-sdk.bat`**: подстановка типичных путей к `dotnet` в `PATH`, без ложной блокировки по `dotnet --list-sdks`/`findstr`.
- **`dev.cmd`**: пункт меню **4** — сборка Release; пункты меню в ASCII (корректное отображение в `cmd` без UTF-8); убрано переопределение `OutputType=Exe` у `dotnet build`/`run` (остаётся **WinExe** из проекта — без лишней консоли).
- **`build-and-pack.ps1`** — перед `dotnet publish` проверка: если запущен `AutoRAW.exe`, выводится предупреждение и скрипт завершается (избежать MSB3027).
- **`bat\_warn-if-autoraw-running.bat`** — вызывается из `Build-AutoRAW.cmd` и `dev.cmd` перед `dotnet build` (предупреждение, не блокирует сборку).
- **Документация**: `LAUNCH-RU.txt` — команды запуска; **Instruction.md** — превью со статусом, ползунок анализа в простом режиме, WebP/JPEG, пути `Товар\webp\` / `Товар\jpg\` при пустой папке выхода, `export_prefs.json`.

### Исправлено
- Сборка/запуск через скрипты больше не принудительно включают консольное окно рядом с WPF.

## [0.4.0] — 2026-05-13

### Добавлено
- **XMP-профиль цветокоррекции.** Вместо ручного ввода числовых параметров теперь загружается `.xmp`-пресет Lightroom / Camera Raw.
  - Профиль «Кроссовки» по умолчанию читает `setting/01.xmp` автоматически.
  - В диалоге редактирования — кнопка «Загрузить .xmp…», открывается сразу в папке `setting/`.
  - После загрузки показывается сводка ключевых параметров: температура, оттенок, экспозиция, контраст, света/тени, насыщенность, резкость и т.д.
  - При создании нового профиля (меню «Профиль → Добавить новый профиль…») тоже можно прикрепить `.xmp`.
- **Расширенная цветокоррекция.** Теперь применяются все ключевые поля XMP: экспозиция (EV), баланс белого, контраст, света, тени, белые/чёрные точки, насыщенность, вибранс, резкость (UnsharpMask), чёткость (Clarity).
- **`build-and-pack.ps1`** — скрипт сборки дистрибутива. Один запуск → папка `dist\AutoRAW\` готова к передаче.
- **`setup.bat`** в дистрибутиве — проверяет наличие .NET 8 Desktop Runtime, при необходимости скачивает (~55 МБ) и устанавливает, затем запускает приложение. При запуске не из папки дистрибутива показывает понятное сообщение об ошибке.

### Изменено
- `ColorCorrectionSettings` переведён в `record` — поддержка `with`-выражений, упрощена работа с копиями.
- `ColorRowDto` и хранилища (`ProfileColorOverrideStore`, `ProductProfileStore`) сохраняют путь к XMP-файлу.
- Сводка цветового профиля на главном экране теперь показывает имя XMP-файла и ключевые параметры.

## [0.3.0] — 2026-05

### Добавлено
- **Цветовой профиль.** Настройки цветокоррекции (контраст, температура, оттенок, тени) для каждого профиля товара. Для «Кроссовок» — значения по умолчанию.
- **Галочка «Применить»** сохраняется отдельно для каждого профиля (файл `profile_prefs.json`).
- **Диалог редактирования цветового профиля** — превью «До» и «После» с живым обновлением.
- **Меню «Профиль»**: встроенный профиль «Кроссовки» и «Добавить новый профиль…». Новый профиль задаёт имя, папки reference/zona и настройки цвета.
- **Меню «Справка»**: «Инструкция» и «Чендж лог» открываются в текстовом окне.
- **`SelectProductCommand`** — выбор профиля из меню с галочкой у активного.

### Изменено
- Простой режим: на главном экране только «Папка товара», «Папка выхода» и блок «Цветовой профиль».
- Папки `reference` и `zona` берутся из профиля товара; по умолчанию — рядом с программой.

## [0.2.0] — 2026-05

### Добавлено
- **Zona-кроп.** Папка `zona` с изображениями, на которых красным прямоугольником обозначена зона кропа. `ZonaCropService` детектирует RotatedRect через `MinAreaRect`, вырезает и поворачивает кадр точно по красной зоне.
- Zona имеет приоритет над reference-кропом.
- Простой / расширенный режим (`IsAdvancedView`).

### Исправлено
- RAW-файлы (`.nef`) теперь декодируются полностью (`DngReadDefines.ReadThumbnail = false`), а не из встроенной миниатюры.
- Корректная авто-ориентация через `AutoOrient` + `ResetPage`.

### Добавлено
