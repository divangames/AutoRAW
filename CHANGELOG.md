# Changelog

Все заметные изменения в этом проекте документируются здесь.
Копия рядом с `AutoRAW.exe` обновляется при каждой сборке через `build-and-pack.ps1`.

**Нумерация версии** (верхняя строка `## [⋯]` задаёт сборку и установщик):

| Позиция | Смысл |
|--------|--------|
| 1 | Тотальный выход релиза |
| 2 | Крупные изменения |
| 3 | Значительные изменения |
| 4 | Простые изменения |
| 5 | Незначительные (в основном фиксы) |

Допускается **1…5** чисел через точку; недостающие справа дополняются нулями (например `0.7.6` → `0.7.6.0.0`). В манифесте Windows и в CLR остаются **первые четыре** компонента; в интерфейсе и в теге релиза — **пять**.

## [0.7.8.5.0] — 2026-05-25

### Добавлено
- **Пакет — «Прогнать через Дроплеты»** (галочка под кнопкой запуска, по умолчанию выкл.): после **полной записи** каждого кадра запускается экшен-дроплет Photoshop из папки **`droplets\`** или **`droples\`** рядом с `AutoRAW.exe` — **`01_drop.exe`** (кадр 01), **`02-03-04-08_drop.exe`** (02, 03, 04, 08), **`05-06-07_drop.exe`** (05–07); в журнал — проверка наличия exe и строка о запуске (pid).
- **Профиль «Кроссовки»:** стартовые ручные правки кадров **05** и **07** в комплекте — `profiles\Sneakers\manual_shot_profile_defaults.json` (переопределяются записями в `%AppData%\AutoRAW\manual_shot_adjust.json`).

### Изменено
- **Редактор — список файлов по подпапкам «Товар»:** рекурсивный обход; деревья **`webp`** / **`jpg`** у **корня** входа скрыты (чтобы не смешивать с RAW); при выборе строки **`webp`** или **`jpg`** — только готовые кадры.
- **Редактор — отзывчивость на слабых ПК:** при перетаскивании результата — коалесценция пересчёта превью (~50 мс), без спиннера поверх кадра; сетка **«Границы (правила макета)»** с первого смещения.
- Настройка дроплетов в **`export_prefs.json`** (`runThroughPhotoshopDroplets`).

### Исправлено
- **Дроплеты Photoshop:** запуск с **`UseShellExecute = false`** и передачей пути к файлу в **Arguments** (раньше при `UseShellExecute = true` аргументы игнорировались — дроплет открывался без файла).
- **Редактор:** повторная подстановка папки «Товар» снова показывает **исходники (RAW)** во вложенных каталогах, а не только `webp`/`jpg`.

### Документация
- **`Instruction.md`**, **`droplets\README.md`** — дроплеты, рекурсия в редакторе, встроенные правки 05/07 для «Кроссовки».

## [0.7.8.2.0] — 2026-05-20

### Добавлено

### Документация

## [0.7.8.0.0] — 2026-05-20

### Важно (поведение)
- **Пакет и редактор:** итоговый кадр строится из **полного исходника** (после AutoOrient и `ShotCropPolicy` по номеру кадра) и **ручных** смещение / масштаб / поворот из редактора / `manual_shot_adjust.json`; **линейный размер выхода** — как у выбранного файла **референса**. Базовое заполнение окна выхода — режим **cover** под пропорции референса (`ManualShotAdjustApplier.ComposeFromFullToReference`).
- **Автоматическое кадрирование по маркёру Zona в пакете отключено** — удалены ветка `ProcessWithZona` и поиск маркёров в `AutoCropBatchService`; одна и та же логика для всех файлов.

### Изменено
- **Главное окно — превью «После»:** один путь без Zona; подпись колонки превью — **«Референс»** (вместо «Референс / технология Zona»).
- **`CropPreviewBitmapFactory`:** единый путь превью; `LoadZonaCroppedPreview*` делегируют в `LoadCroppedPreview*`; добавлен **`TryLoadPreparedFullForManualFrame`** для кэша полного кадра в редакторе.
- **`CanRunBatch`:** папка **`reference`** и выбранный референс для строки **обязательны всегда** (раньше при наличии только `zona` референс можно было не задавать).
- **Редактор положения кадра:** прокрутка окна, кнопки в стилях главного окна (`CommonButtons.xaml`); превью без участия Zona в кропе; сетка PNG из `zona` по-прежнему только для подсказки на превью.
- **`Instruction.md`:** в начале раздела про Zona зафиксировано отключение автокропа; обновлены редактор, пакет, таблица выбора кадра, расширенный режим.

### Исправлено
- **`CropPreviewBitmapFactory.LoadCroppedPreviewWithSize`:** инициализация `working` перед `finally` (CS0165).

- **Тёмная тема:** доработаны меню, кнопки, границы, текст, слайдер и полосы прокрутки (`Themes/Dark*.xaml`).

- **Сборка:** невалидный второй стиль `ScrollBar` в `DarkInteractive.xaml`; порядок объявления `CancelBatchButtonStyle` относительно `SecondaryButtonStyle` в XAML.
