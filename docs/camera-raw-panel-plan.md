# Панель «Редактировать» (Camera Raw / Photoshop) — план

Референс: боковая панель Adobe Camera Raw / Lightroom Develop — гистограмма, профиль, секции **Свет**, **Цвет**, **Эффекты**, слайдеры с числом справа, градиентные дорожки (температура, оттенок).

**Связано с темой UI:** [`dark-theme-backlog.md`](dark-theme-backlog.md) (тёмная палитра, кастомные слайдеры).  
**Чат Zona:** [`zona-chat-status.md`](zona-chat-status.md) (отдельно, на паузе).

---

## Что уже есть в коде (база)

| Компонент | Файл | Состояние |
|-----------|------|-----------|
| Модель параметров LR-подобная | `Models/ColorCorrectionSettings.cs` | Exposure, Contrast, Highlights, Shadows, Whites, Blacks, Temp, Tint, Vibrance, Saturation, Clarity, Dehaze, Sharpness |
| Применение к изображению | `Services/ColorCorrectionService.cs` | Magick.NET после кропа |
| Импорт .xmp | `Services/XmpSettingsParser.cs` | Загрузка пресета |
| Редактор профиля | `ColorProfileEditorDialog.xaml` | До/После + XMP, **без** панели слайдеров как в CR |
| В главном окне | `MainWindow.xaml` | Блок «Цветовой профиль» (вкл/выкл, кратко) |

**Вывод:** логика цвета и XMP частично готова; не хватает **UX панели разработки** (как на скрине).

---

## Целевой UI (из референса)

1. **Верх:** гистограмма (RGB), кнопки Auto / B&W / HDR (опционально, фаза 2–3).
2. **Профиль:** выпадающий список + сетка пресетов.
3. **Свет:** Экспозиция, Контрастность, Светлые, Тени, Белые, Чёрные.
4. **Цвет:** Баланс белого (как снято + пипетка), Температура, Оттенок, Красочность, Насыщенность.
5. **Эффекты:** Текстура, Чёткость, Удаление дымки, Виньетка, Зерно (последние — позже; в модели пока нет Texture/Vignette/Grain).

Секции — **сворачиваемые** (`Expander`), одна колонка, тёмный фон `#1E1E1E` / `#252526`.

## Варианты реализации (что выбрать)

### Вариант A — Чистый WPF (рекомендуемый старт)

**Суть:** свои `UserControl` + стили в `Themes/`, без смены стека.

| Плюсы | Минусы |
|-------|--------|
| Уже есть `Theme.*`, `DarkInteractive` (Slider) | Градиентные слайдеры (Temp/Tint) — свой `ControlTemplate` |
| Полный контроль, один .exe | Гистограмма — отдельная отрисовка |
| Стыкуется с `ColorCorrectionSettings` | Объём XAML на 1–2 итерации |

**Состав:**
- `DevelopSliderControl` — подпись | слайдер | `TextBox` (число), Min/Max/Step, привязка к `double`.
- `DevelopSectionControl` — заголовок + `Expander` + список слайдеров.
- `DevelopPanelViewModel` — обёртка над `ColorCorrectionSettings`, `INotifyPropertyChanged`, команды Reset/Auto (позже).
- Превью: debounce 150–300 ms → `ColorCorrectionService` → обновление `PreviewAfter` (как в редакторе профиля).

**Куда встроить:**
- **A1.** Переделать `ColorProfileEditorDialog` — быстрый win.
- **A2.** Боковая колонка в `MainWindow` (расширенный режим), как CR — основной UX.

---

### Вариант B — WPF + готовые UI-библиотеки

| Библиотека | Что даёт | Заметка |
|------------|----------|---------|
| **WPF UI** (lepoco) | Современные слайдеры, NumberBox, Card | NuGet; стиль ближе Win11, нужна стыковка с `Theme.*` |
| **MaterialDesignInXaml** | Slider, Expander, темы | Тяжелее, другой визуальный язык |
| **HandyControl** | Slider, NumericUpDown | Популярен в CN-сообществе |
| **MahApps.Metro** | Хром окон + контролы | Избыточен, если Metro не нужен |

**Когда имеет смысл:** если вариант A затянется из‑за слайдеров; взять **WPF UI** только для `NumberBox` + базового Slider, градиент всё равно кастом.

---

### Вариант C — Гистограмма и «тяжёлая» графика

| Подход | Описание |
|--------|----------|
| **OxyPlot** / **LiveCharts2** | Гистограмма по данным из Magick/OpenCV |
| **WriteableBitmap + свой код** | Подсчёт bins по RGB после загрузки превью |
| **OpenCvSharp** (уже в проекте) | `CalcHist` для превью-кадра |

Рекомендация: **фаза 3** — сначала слайдеры и live preview, гистограмма после.

---

### Вариант D — Обработка превью в реальном времени

| Движок | Уже в проекте | Роль |
|--------|---------------|------|
| **Magick.NET** | ✅ `ColorCorrectionService` | Основной pipeline, batch + превью |
| **OpenCvSharp** | ✅ кроп/анализ | Опционально ускорить resize/preview |
| **GPU (OpenCL/CUDA)** | ❌ | Не нужно на первом этапе |

**Паттерн:** слайдер двигается → `DispatcherTimer` debounce → копия превью-буфера → `ApplyIfEnabled` → обновить `Image.Source`.

Панель CR в WinUI проще визуально (NumberBox, анимации), но **отдельный большой проект**: новый shell, перенос MVVM, installer.

См. бэклог в `dark-theme-backlog.md` → пункт про слайдер с заливкой пройденного участка.
