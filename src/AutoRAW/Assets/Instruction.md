# AutoRAW — инструкция

> **Справка → Инструкция** открывает этот файл с оформлением Markdown (как на GitHub): заголовки, списки, код, таблицы. Файл `Instruction.md` кладётся рядом с `AutoRAW.exe` при сборке — его можно править под свою раскладку.

---

## Простой режим (по умолчанию)

1. В меню **Профиль** выберите нужный профиль (по умолчанию — **Кроссовки**). Папки `reference` и `zona` берутся **рядом с программой**, если в расширенном режиме не заданы другие пути.
2. **Папка «Товар»** — каталог с исходными снимками (например `.nef`).
3. **Папка выхода** — необязательна. Если оставить пустой, для каждого исходника создаётся подпапка **внутри папки «Товар»**: при WebP — каталог `webp`, при JPEG — `jpg`. Имя файла совпадает с исходником по основе имени, расширение `.webp` или `.jpg` (без суффикса `_crop`).
4. Под кнопкой **Запустить кадрирование** — галочка **«Сохранять результат в WebP»**: включена по умолчанию → выход **WebP**; снята → **JPEG**, как раньше. Состояние запоминается в `%AppData%\AutoRAW\export_prefs.json`.
5. Нажмите **Запустить кадрирование**.

### Превью и ползунок «Анализ»

- Пока пересчитывается превью (**До** / **После**), над блоком показываются **текст статуса** и **полоска прогресса** — так видно, что программа не зависла (в т.ч. при переключении **Вид → Расширенный режим** и обратно).
- В группе **Превью** есть ползунок **«Анализ (длинная сторона)»** — тот же параметр, что в расширенном режиме: влияет на расчёт кропа по референсу. При движении ползунка превью снова пересчитывается **с тем же индикатором загрузки**.

### Как выбирается кадр

| Условие | Поведение |
|--------|------------|
| В `zona` есть файл с тем же именем, что и входной, с **красной рамкой** | Кадр по этой рамке (включая поворот) |
| Иначе | Кроп по **референсам** из папки `reference` |

Папка `zona` может быть рядом с программой или в пакете профиля (`user files\Profile\…`).

## Расширенный режим

**Вид → Расширенный режим** — таблица соответствий файлов, превью, ручные пути, ползунок анализа, журнал (по умолчанию скрыт: **Вид → Показать журнал**). У превью (**Референс / Zona · До · После**) при пересчёте тоже отображаются **статус и полоска загрузки**. Галочка **WebP / JPEG** дублируется в шапке под ползунком анализа (одна и та же настройка, что в простом режиме).

---

## Свой профиль

**Профиль → Добавить новый профиль…** — укажите папку, внутри которой есть подкаталоги `reference` и `zona`, задайте имя. Профиль сохраняется в **`user files\Profile`** и появляется в меню **Профиль**.

**Профиль → Перейти в мой каталог профилей…** — открывает `%LocalAppData%\AutoRAW\user files\Profile` в Проводнике (можно вручную копировать или править папки профилей).

## Справка в меню

| Пункт | Файл |
|-------|------|
| **Инструкция** | `Instruction.md` рядом с программой |
| **Чендж лог** | `CHANGELOG.md` (из репозитория, копируется при сборке) |
| **О программе…** | версия, команда **WhiteCube**, разработчик |

### Формат файлов после кадрирования

| Папка выхода | Галочка WebP | Куда и как сохраняется |
|--------------|----------------|------------------------|
| **Не задана** | Включена (по умолчанию) | `<Папка товара>\webp\<имя как у исходника>.webp` |
| **Не задана** | Снята | `<Папка товара>\jpg\<имя как у исходника>.jpg` |
| **Задана** | Включена | `<Папка выхода>\<имя>.webp` |
| **Задана** | Снята | `<Папка выхода>\<имя>.jpg` |

Пример: вход `D:\фото\01.nef`, выход пустой, WebP → `D:\фото\webp\01.webp`.
