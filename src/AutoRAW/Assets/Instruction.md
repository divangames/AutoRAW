# AutoRAW — инструкция

> **Справка → Инструкция** открывает этот файл с оформлением Markdown (как на GitHub): заголовки, списки, код, таблицы. Файл `Instruction.md` кладётся рядом с `AutoRAW.exe` при сборке — его можно править под свою раскладку. Строка **«Версия»** в программе — **пятизначная** (`0.7.8.5.0`); подробнее о позициях чисел см. **Справка → Чендж лог** (таблица в начале `CHANGELOG.md`).

## Semi-AI: эталон `reference\` → подгонка RAW (основной сценарий)

**Эталон** — готовые кадры в `reference\Sneakers\01.jpg` … `08.jpg`, где товар уже стоит как нужно.

**Шаг 1 — анализ эталона (при загрузке профиля / смене reference):** для каждого `NN.jpg` программа **запоминает шаблон** композиции:

| Параметр | Смысл |
|----------|--------|
| Размер кадра | пиксели референса (например 1400×1050) |
| Центр товара | доли ширины и высоты кадра |
| Размер товара | ширина и высота bbox в долях кадра |
| Отступы | L / R / T / B от bbox до краёв кадра |

Детекция на эталоне и на RAW — **YOLOv8-seg** (маска силуэта) → **YOLOv8** → **OpenCV**; **U2Net-p** уточняет bbox после YOLO-detect. Модели: `models/subject/` (`bat/download-yolov8n-seg-onnx.ps1` и др.).

**Шаг 2 — другой снимок (например `test\RAW\46741\8.NEF`):** находит товар на полном RAW, **двигает**, **масштабирует**, **центрирует** так, чтобы в выходе 1400×1050 композиция совпала с шаблоном `reference\08.jpg`.

**Шаг 3 — превью:** на главном экране **«После (авто)»** и **«После (экспорт)»** показывают, как будет кадр по шаблону референса; в редакторе — крупное превью + ручная доработка.

**Важно:** папка **`zona\Sneakers\operation`** в этом сценарии **не используется** (это старый макетный пайплайн Zona). Для подгонки нужны только **`reference\Sneakers`** и исходники RAW.

## Технология «Zona» (исторически; автокроп отключён)

**Текущее поведение программы:** пакет и редактор строят кадр из **полного** исходника и параметров смещения / масштаба / поворота; размер выхода = **референс**. **Красные линии** на превью редактора рисуются **программой** по правилам макета (не из `zona_tovara_*.png` и не из `NN_line.png`). **Авто-подгонка** для 01–04, 06, 08: центр товара между вертикалями, низ у нижней линии, товар не выходит за границы.

**Смысл кадров 01–08 (профиль «Кроссовки»):**

| Кадр | Содержание кадра |
|------|------------------|
| **01** | Главная (обложка): пара на **¾**, слегка развёрнута |
| **02** | Сбоку — **наружная** сторона |
| **03** | Сбоку — **внутренняя** сторона |
| **04** | Пятки, **¾ сзади** |
| **05** | Стельки крупно (**вид сверху** на стельки) |
| **06** | Пара кроссовок **вид сверху** |
| **07** | Крупно шнурки и язычки |
| **08** | Подошва |

**Авто-подгонка** (bbox **U2Net-p** / OpenCV; макет **1400×1050**, масштаб от референса). На **главном окне** глубину анализа задаёт ползунок **«Анализ (длинная сторона)»**. В окне **редактора** то же дерево расчётов и **те же число проходов уточнения**, что и в пакете; для отзывчивости интерфейса длинная сторона входа для кнопки «Авто-подгонка» **ограничена сверху** (не выше ~**1280 px**), если в главном окне выставлено больше.

| Кадр | Правила на холсте |
|------|-------------------|
| **01** | ширина товара **965 px**, слева **223 px**, снизу **185 px** |
| **02, 03, 04, 08** | ширина **965 px**, слева **223 px**, снизу **265 px** |
| **06** | высота **897 px**, отступ **76 px** сверху и снизу, **центр по X** (ширина пропорционально) |
| **05, 07** | только **вручную** (авто-подгонка отключена) |

Если детектор «видит» только часть кроссовка (пятка, фрагмент подошвы), авто-подгонка **не применяется** — вместо зума 200% и увода за край появится сообщение, что результат ненадёжен; подгоните вручную.

**Редактор кадра:** **↶ / ↷** — отмена и повтор правок; в статусе авто-подгонки — провайдер ONNX (**DirectML** или **CPU**). Центр товара — по **силуэту** (не только bbox), угол **PCA** показывается в деталях детекции.

Для **01–04** и **08** в коде используется общий режим «широкий товар» (проверка ракурса: объект шире, чем выше). Для **06** — отдельный режим «высокий товар» (пара сверху, выше, чем шире). Это **не** означает, что все перечисленные кадры сняты сверху — только **05** и **06** в смысле ракурса «сверху».

**Технология «Zona»** в исходном виде — **собственная разработка для AutoRAW**: модуль кадрирования по цветному маркёру на изображении; ниже — описание для справки и совместимости структуры профиля.

**Сценарий operation (профиль «Кроссовки»).** Если есть папка `zona\Sneakers\operation\02\` (номер кадра `02` или `2`, файл `2.NEF` → `02`):

| Файл в папке | Назначение |
|--------------|------------|
| `01_center.png` | Зелёный прямоугольник — объект; красное перекрестие — куда ставить центр товара |
| `02_crop.png` | Красная рамка — **зона кадрирования** (можно только три линии: лево, право, низ — без верхней; высота рамки берётся из соотношения сторон референса) |
| `03_crop.png` | Пример результата кропа (для сверки, программа не читает) |
| `04_final_1400x1050.png` | Пример финального размера (для сверки; экспорт берёт **фактические** пиксели из `reference\NN` для этого номера) |

Если папки `operation\NN\` **нет**, этот сценарий **не** применяется: кадр обрабатывается как обычно (**Zona** по маркёру `zona\NN.*` или кроп по референсу).

Для **operation** не выполняется автоматический **поворот кадра «03»** (`ShotCropPolicy`), который включается для обычного Zona у высоких объектов — макеты `01_center` / `02_crop` должны совпадать с **той же ориентацией**, что и RAW после **AutoOrient** (без этого поворота).

Доли для кропа и для зоны объекта считаются в **одной** системе: полноразмерный **`01_center.png`**. Если **`02_crop.png`** другого размера, рамка переносится линейно в холст центрирования. Для operation не подмешиваются **`NN_line.png`** из корня `zona` — только макеты из `operation\NN\`.

Масштаб и положение перед рамкой `02_crop` считаются по **отдельной** оценке контура товара (`EstimateForOperation`), не по общему пайплайну line-guide.

Если красную рамку из собственного `02_crop.png` разобрать не удаётся, программа пробует общий **`zona_tovara.png`** в корне той же папки `zona` (пара лево–право–низ — без верхней линии).

**Направляющие для центрирования (профиль «Кроссовки»).** В `zona\Sneakers` могут лежать файлы `01_line.png`, `02_line.png`, … — красные линии границ кадра. **Масштаб** кадра задаёт маркёр Zona (`01.jpg`, …) или референс; **позицию** (центр между вертикальными линиями, низ по нижней) — шаблон `_line.png`. Область кропа считается в **оригинале** до обрезки, без сдвига уже готового кадра. Если `07_line.png` нет — для кадра 07 используется обычное выравнивание.

**Как это работает.** В профиле задаётся каталог с именем `zona` (так принято в структуре профиля рядом с `reference` и `setting`). В нём лежат **маркёрные** снимки: на них **красным** обведена нужная область кадра. Для каждого входного файла программа ищет в `zona` файл с **тем же именем** (другое расширение допускается). Если маркёр найден и красная зона распознана, **Zona** вычисляет геометрию рамки (в т.ч. поворот), переносит её на полноразмерный исходник и **вырезает кадр** с учётом угла. Если маркёра нет или зона не найдена — используется кроп по **референсу** из `reference`. При наличии валидного маркёра **Zona имеет приоритет** над референсом.

**Зачем отдельное имя.** Алгоритмы детекции и геометрии заточены под сценарий AutoRAW (цвет маркёра, связка с RAW/JPEG, превью и экспорт). В интерфейсе и документации мы называем этот режим **технология «Zona»**; папка на диске по-прежнему называется `zona` — это часть контракта профиля.

*Этот раздел обновляется вместе с продуктом; при изменении поведения Zona правьте его же в репозитории (`src/AutoRAW/Assets/Instruction.md`).*

## Главное окно

1. В меню **Профиль** выберите профиль: **Стандартные** (по умолчанию — **Кроссовки**, плюс каталоги из `profiles\` рядом с программой) или **Пользовательские** (ваши пакеты в `%LocalAppData%\AutoRAW\user files\Profile`). Для **Кроссовок** референсы и каталог **Zona** лежат в **`reference\Sneakers`** и **`zona\Sneakers`**. У других профилей — своя подпапка с тем же именем, что и профиль (`reference\<имя>\`, `zona\<имя>\`). Свои каталоги reference/zona задаются при **добавлении пользовательского профиля** (структура папки профиля). Сопоставление кадров по режимам «Стандартный / Руслан / Маша» в коде сохранено; пункт **«Фотограф»** в меню временно скрыт.
2. **Папка «Товар»** — каталог с исходными снимками (включая **все вложенные подпапки**). **Стандартный:** в каждой папке файлы по порядку имени — 1-й→`01`, 2-й→`02`, … 8-й→`08`. **Руслан:** номер из имени (`1.NEF`, `тип3`…). **Маша:** по совпадению имени с референсом. Каталоги `webp` и `jpg` внутри входа не обрабатываются повторно. Папку можно выбрать через **Обзор…** или **перетащить** в поле «Товар (вход)».
3. **Папка выхода** — необязательна.
   - **Пустая** — создаётся `webp` или `jpg` **в корне «Товар»**, внутри — те же подпапки, что во входе: `Товар\webp\партия_A\01.webp`.
   - **Задана** — в неё копируется та же структура подпапок: `D:\готово\партия_A\01.webp`.
4. Кнопка **«Запустить кадрирование»** во время работы становится **«Пауза»** / **«Продолжить»**; узкая **«Отмена»** справа прерывает пакет.
5. Галочка **«Сохранять результат в WebP»** (по умолчанию включена) и **«Прогнать через Дроплеты»** (по умолчанию выключена) хранятся в `%AppData%\AutoRAW\export_prefs.json`. При включении дроплетов после **полной записи** каждого кадра пакета программа запускает `.exe` из **`droplets\`** или **`droples\`** рядом с программой и передаёт путь к сохранённому WebP/JPEG.

   | Номер кадра (экспорт) | Дроплет |
   |----------------------|---------|
   | **01** | `01_drop.exe` |
   | **02, 03, 04, 08** | `02-03-04-08_drop.exe` |
   | **05, 06, 07** | `05-06-07_drop.exe` |

   Номер кадра — как в очереди (режим фотографа: порядок в папке, номер в имени файла или сопоставление с референсом). Галочка **«Открывать NEF/RAW через LibRaw»** — `%AppData%\AutoRAW\raw_loader_prefs.json` (при сбое LibRaw — ImageMagick).
6. Над журналом — **полоса прогресса** (`94/100`, при паузе — «На паузе», по завершении — «Готово 100/100»).
7. **Журнал**: **⧉** — отдельное окно без системной шапки (перетаскивание за заголовок «Журнал», изменение высоты); повторное **⧉** — снова внизу главного окна. **✕** — скрыть. Цвета строк: ошибки и отмена — красный, пауза — синий, итог «Готово!» — зелёный.

### Превью и ползунок «Анализ»

- Пока пересчитывается превью (**Референс · До · После (авто) · После (экспорт)**), над блоком показываются **текст статуса** и **полоска прогресса**, а внутри каждой ячейки превью — **круглый индикатор загрузки**, пока картинка не готова. Блок превью можно скрыть: **Вид → Окна → Показать превью**.
- В группе **Превью** есть ползунок **«Анализ (длинная сторона)»** — он влияет на расчёт кропа по референсу. При движении ползунка превью снова пересчитывается **с тем же индикатором загрузки**. После правок в **Редакторе положения кадра** колонка **После** на главном экране обновляется вместе с сохранением правок в json (в том числе при перетаскивании результата мышью — без паузы на экране редактора).
- **Вид → Редактор положения кадра…** — отдельное окно: **дерево папок** «Товар» (корень с именем входной папки, вложенные каталоги раскрываются). **Корень** + режим **«Все файлы»** — все снимки во всём дереве, **с заголовками подпапок**; отдельная подпапка (например `47133`) — только её файлы. **«По номеру / имени»** — в поле введите `1`, `2`, `01` (номер кадра по правилам пакета) или часть имени. Миниатюры списка — **лёгкие (~96 px, JPEG)** с превью кадра и **красной сеткой** правил макета (без ползунка «Анализ» главного окна). Опционально **«Группировать по имени файла»** (иначе при нескольких папках — группировка по каталогам). Имя файла — **двойной щелчок** для переименования на диске (json пропусков и правок подстраиваются). **Галочка «в очередь»** у каждого файла — см. `%AppData%\AutoRAW\editor_skipped_files.json`. По превью миниатюры — клик **открывает** режим правки (без Ctrl; при Ctrl/Shift выделение строк как обычно). У списка папок — **«Пропустить»** / **«Вернуть»** (`editor_ignored_folders.json`). В режиме правки большие превью «Референс» и «Результат» показывают **круглый индикатор загрузки** на время расчёта (**не** показывается поверх «Результата» во **время активного перетаскивания** мышью — чтобы на слабых ПК кадр не «дёргался»; расчёт при перетаскивании ограничивается по частоте ~**50 мс** на движение; **красная сетка правил макета** уже с первого смещения, если в комбобоксе выбрано **«Границы (правила макета)»**; профильная **цветокоррекция** на это превью — после отпускания кнопки или паузы). Меню **Файл → Применить профиль**: только эта фотография (json), ко всем файлам с **тем же именем** в текущем списке браузера (плюс **постоянное** правило «профиль + имя файла» без пути в `manual_shot_adjust.json` для любых других папок), пункт **«Убрать правило профиля по имени текущего файла»** удаляет только эту привязку по имени. **Файл → Сохранить для профиля (текущий кадр)** — умолчание по **номеру кадра** и одновременно по **имени текущего файла**. **Файл → Сохранить как профиль…** создаёт отдельный bundle. Кнопка **«Авто-подгонка»** (и автоматический запуск при открытии файла без сохранённых правок) подбирает смещение и масштаб **по референсу** (детекция **YOLOv8** → **OpenCV**, затем уточнение по собранному кадру). Макеты **operation/01_center** в редакторе **не** используются — они для отдельного сценария Zona. Модель: `bat/download-yolov8n-onnx.ps1`. Справа кнопки **«Применить изменения»** (явное сохранение только для этого файла) и **«Сбросить»** (убрать per-file переопределение и подставить настройки профиля по кадру). Текст подсказки внизу окна на несколько секунд подтверждает сохранение и другие действия. Для скорости **исходный файл в режиме правки загружается с уменьшением до длинной стороны не выше ~600 px** (только для превью в этом окне; пакет и главный экран по-прежнему используют полный кадр согласно **«Анализ»**). Превью «Результат» во время движения ползунков считается **в черновом** разрешении; после паузы — уже не выше **~600 px** по длинной стороне (рамка блока результата **до ~600×450**, пропорция как у эталона). В режиме правки масштаб и смещения задаются в **координатах финального экспорта** (пиксели референса, например 1400×1050 для «Кроссовки»), а не по пикселям маленькой картинки на экране. **Сетка выравнивания** из папки `zona` — только на экране. При закрытии окна редактор по-прежнему **автоматически** записывает текущие ползунки для открытого файла в per-file слот.
- По выделенным в редакторе строкам-подпапкам **«Товар»** файл в списке миниатюр ищется **рекурсивно** (видны RAW/исходники в любом подкаталоге). Деревья **`…\webp\`** и **`…\jpg\`**, которые начинаются **в корне папки «Товар»** (стандартный вывод программы без отдельного «Выход»), среди них **не показываются**, чтобы не смешиваться с исходниками; если нужны только готовые кадры — отметьте в дереве строку **`webp`** или **`jpg`**.
- Когда в редакторе **собран список файлов** по выбранным подпапкам, **в фоне** для снимков без сохранённых per-file правок, без **правила профиля по имени файла**, и без **ненулевой** подстройки профиля по номеру кадра запускается та же авто-подгонка по эталону, что при нажатии «Авто-подгонка»; успешные результаты пишутся в `manual_shot_adjust.json` **одним** сохранением, чтобы при **«Запустить кадрирование»** уже не считать силуэт повторно. Пока пакет на главном экране **выполняется** (кнопка «Пауза», прогресс), фон **ждёт** — декод RAW и загрузка диска не конкурируют с очередью. Кадры **05 и 07**, файл открытый в режиме правки и пути без пары в `reference\` не обрабатываются в этом проходе. **При «Запустить кадрирование»** наоборот: пакет **ждёт** завершения фоновых расчётов **глифов очереди** и этого прохода **авто-подгонки** (показ в журнале строки про ожидание), чтобы не перегружать машину параллельными тяжёлыми задачами.
- На главном экране **«После (авто)»** — только авто-подгонка; **«После (экспорт)»** — то, что уйдёт в пакет: сохранённые правки (**per-file**, правило профиля по **имени файла** без пути, правило профиля по **номеру кадра** с реальной подстройкой, затем авто-подгонка). После сохранений в редакторе (и при действиях «только этот файл» / «ко всем с тем же именем» и т.п.) привязка **номера эталона** к **полному пути** входной фотографии дополнительно пишется в `%AppData%\AutoRAW\crop_frame_refs.json` — чтобы при исключении части файлов из очереди (галочки, пропуски) **режим фотографа «Стандартный» не переименовывал первый оставшийся снимок в `01`** вместо выбранного кадра. **Тот же файл обновляется сразу при смене номера кадра в комбобоксе редактора** (не нужно только двигать слайдеры или сохранять правки через меню — достаточно выбрать 02 / 04 / 08 перед пакетом). Запись в профиле по номеру кадра только с «нулями» не блокирует авто-подгонку; на экспорт может подставляться только **сетка** превью из профиля или по имени. Под колонками — подпись **«Экспорт: …»**.
- Блок **«Очередь»** на главном экране: у каждого файла **✓** (сохранённые правки), **⚠** (авто или низкая оценка качества), **✗** (ошибка). Ползунок **«Порог %»** (по умолчанию 68, файл `%AppData%\AutoRAW\align_quality_prefs.json`) — ниже порога в пакете и в списке будет ⚠. Галочка **«Только ⚠»** фильтрует очередь.
- **Пакет «Запустить кадрирование»** использует ту же логику, что **«После (экспорт)»**. В журнале: **OK ✓** — сохранённые правки; **OK ⚠ авто** — авто-подгонка; **OK ⚠ 62%** — низкая оценка совпадения с референсом. В конце — сводка: ✓ / ⚠ авто / низкое качество / проверить.
- **`models/subject/yolov8n-seg.onnx`** — сегментация (приоритет); **`yolov8n.onnx`**, **`u2netp.onnx`** — см. `models/subject/README.md`.
- После сохранения на **превью** и при **пакетном экспорте** используется одна и та же геометрия: **полный** кадр исходника → **смещение / масштаб / поворот** → **выход** строго в размере выбранного **референса**; **сетка** на диск не попадает.

### Как выбирается кадр

Все входные файлы обрабатываются **одинаково**: размер и пропорции результата задаёт файл из папки `reference` для выбранного номера кадра (01…08); положение и масштаб товара — **ручные** настройки редактора / json. Папка `zona` опциональна (**PNG сетки** для редактора).

Если профиль перешёл в состояние черновика **«Несохранённые изменения»** (например после смены путей или цветокоррекции относительно исходного пакета), сохраните его через меню **Профиль**: **Применить черновик к исходному профилю…** или **Сохранить черновик как новый профиль…**.

## Свой профиль

**Профиль → Добавить новый профиль…** — укажите **папку профиля** (готовый bundle с `profile.json`, каталог из `profiles\<slug>` проекта, или папки `reference` / `zona`) и **имя** (можно оставить предложенное из имени папки). Профиль **копируется** в `%LocalAppData%\AutoRAW\user files\Profile\<имя>\` (`reference\<имя>\`, `zona\<имя>\`, `setting`, `profile.json`).

Штатная поставка включает `profiles\Sneakers\manual_shot_profile_defaults.json`: **стартовые** смещение и масштаб (**кадры 05 и 07**) для профиля **«Кроссовки»** до любых сохранений в `%AppData%\AutoRAW\manual_shot_adjust.json`; ваши сохранённые правила профиля и per-file **перекрывают** этот файл.

**Профиль → Перейти в мой каталог профилей…** — открывает `%LocalAppData%\AutoRAW\user files\Profile` в Проводнике (можно вручную копировать или править папки профилей).

## Профили в комплекте программы (сборка из репозитория)

Чтобы пресет попал **ко всем пользователям** в меню «Профиль» вместе с установщиком, в корне репозитория заведите:

1. **`reference\<slug>\`** и **`zona\<slug>\`** — те же данные, что для пользовательского профиля (`slug` — короткое имя папки на латинице, без пробелов).
2. **`profiles\<slug>\profile.json`** — манифест с полем **`displayName`** (имя в меню, можно по-русски) и **`color`** (как в `profile.json` у профиля в `%LocalAppData%`).

При `dotnet build` / `dotnet publish` содержимое **`profiles\`** копируется рядом с `AutoRAW.exe`. Профили из этого каталога появляются в меню **Профиль → Стандартные**. Профиль «Кроссовки» задаётся через **`profiles\Sneakers\`** и папки **`reference\Sneakers`**, **`zona\Sneakers`**.

## Уведомления в Telegram

**Настройки → Telegram ZONA…** — при **успешном завершении пакета**, **ошибке** и **отмене** ZONA может отправить сообщение в ваш чат (токен бота от @BotFather и ID чата). Настройки сохраняются в `%AppData%\AutoRAW\telegram_zona.json`. После сохранения токен и chat id можно сменить только через **«Изменить…»** и пароль.

Сообщение о **готовом пакете** выглядит так: сначала короткая фраза от ZONA, затем строка **✅ Успешно: N, ошибок: M, всего: K. Время: …**, затем **Профиль: имя** — без отдельной служебной первой строки «пакет готов».

### Формат файлов после кадрирования

| Папка выхода | Галочка WebP | Куда и как сохраняется |
|--------------|----------------|------------------------|
| **Не задана** | Включена (по умолчанию) | `<Папка товара>\webp\<подпапка>\*.webp` |
| **Не задана** | Снята | `<Папка товара>\jpg\<подпапка>\*.jpg` |
| **Задана** | Включена | `<Папка выхода>\<подпапка>\*.webp` |
| **Задана** | Снята | `<Папка выхода>\<подпапка>\*.jpg` |

Пример: вход `D:\фото\партия\01.nef`, выход пустой, WebP → `D:\фото\webp\партия\01.webp`.

Готовые кадры по профилю «Кроссовки» приводятся к размеру **референса** (например **1400×1050** px).
