# AutoRAW — инструкция

> **Справка → Инструкция** открывает этот файл с оформлением Markdown (как на GitHub): заголовки, списки, код, таблицы. Файл `Instruction.md` кладётся рядом с `AutoRAW.exe` при сборке — его можно править под свою раскладку. Строка **«Версия»** в программе — **пятизначная** (`0.7.8.2.0`); подробнее о позициях чисел см. **Справка → Чендж лог** (таблица в начале `CHANGELOG.md`).

## Технология «Zona» (исторически; автокроп отключён)

**Текущее поведение программы:** пакет **«Запустить кадрирование»** и **редактор положения кадра** строят результат из **полного** исходника (после AutoOrient и `ShotCropPolicy` по номеру кадра) и **ручных** параметров смещения / масштаба / поворота (`manual_shot_adjust.json`); линейный размер выхода совпадает с **референсом** (`reference\NN`). **Автоматическое** вырезание по красному маркёру в папке `zona` **не** используется. Папка `zona` по-прежнему может содержать PNG **сетки выравнивания** для подсказки в редакторе (`zona_tovara_01.png` и т.д.).

---

**Технология «Zona»** в исходном виде — **собственная разработка для AutoRAW**: модуль кадрирования по цветному маркёру на изображении; ниже — описание для справки и совместимости структуры профиля.

**Сценарий operation (профиль «Кроссовки»).** Если есть папка `zona\Sneakers\operation\02\` (номер кадра `02` или `2`, файл `2.NEF` → `02`):

| Файл в папке | Назначение |
|--------------|------------|
| `01_center.png` | Зелёный прямоугольник — объект; красное перекрестие — куда ставить центр товара |
| `02_crop.png` | Красная рамка — **зона кадрирования** (можно только три линии: лево, право, низ — без верхней; высота рамки берётся из соотношения сторон референса) |
| `03_crop.png` | Пример результата кропа (для сверки, программа не читает) |
| `04_final_1400x1050.png` | Пример финального размера (для сверки; экспорт берёт **фактические** пиксели из `reference\NN` для этого номера) |

Если папки `operation\NN\` **нет**, этот сценарий **не** применяется: кадр обрабатывается как обычно (**Zona** по маркёру `zona\NN.*` или кроп по референсу).

Для **operation** не выполняется автоматический **поворот кадра «03»** (`ShotCropPolicy`), который включается для обычного Zona у высоких объектов — макеты `01_center` / `02_crop` должны совпадать с **той же ориентацией**, что и RAW после **AutoOrient** (без этого поворота).

Доли для кропа и для зоны объекта считаются в **одной** системе: полноразмерный **`01_center.png`**. Если **`02_crop.png`** другого размера, рамка переносится линейно в холст центрирования. Для operation не подмешиваются **`NN_line.png`** из корня `zona` — только макеты из `operation\NN\`.

Масштаб и положение перед рамкой `02_crop` считаются по **отдельной** оценке контура товара (`EstimateForOperation`), не по общему пайплайну line-guide.

Если красную рамку из собственного `02_crop.png` разобрать не удаётся, программа пробует общий **`zona_tovara.png`** в корне той же папки `zona` (пара лево–право–низ — без верхней линии).

**Направляющие для центрирования (профиль «Кроссовки»).** В `zona\Sneakers` могут лежать файлы `01_line.png`, `02_line.png`, … — красные линии границ кадра. **Масштаб** кадра задаёт маркёр Zona (`01.jpg`, …) или референс; **позицию** (центр между вертикальными линиями, низ по нижней) — шаблон `_line.png`. Область кропа считается в **оригинале** до обрезки, без сдвига уже готового кадра. Если `07_line.png` нет — для кадра 07 используется обычное выравнивание.

**Как это работает.** В профиле задаётся каталог с именем `zona` (так принято в структуре профиля рядом с `reference` и `setting`). В нём лежат **маркёрные** снимки: на них **красным** обведена нужная область кадра. Для каждого входного файла программа ищет в `zona` файл с **тем же именем** (другое расширение допускается). Если маркёр найден и красная зона распознана, **Zona** вычисляет геометрию рамки (в т.ч. поворот), переносит её на полноразмерный исходник и **вырезает кадр** с учётом угла. Если маркёра нет или зона не найдена — используется кроп по **референсу** из `reference`. При наличии валидного маркёра **Zona имеет приоритет** над референсом.

**Зачем отдельное имя.** Алгоритмы детекции и геометрии заточены под сценарий AutoRAW (цвет маркёра, связка с RAW/JPEG, превью и экспорт). В интерфейсе и документации мы называем этот режим **технология «Zona»**; папка на диске по-прежнему называется `zona` — это часть контракта профиля.

*Этот раздел обновляется вместе с продуктом; при изменении поведения Zona правьте его же в репозитории (`src/AutoRAW/Assets/Instruction.md`).*

## Главное окно

1. В меню **Профиль** выберите профиль: **Стандартные** (по умолчанию — **Кроссовки**, плюс каталоги из `profiles\` рядом с программой) или **Пользовательские** (ваши пакеты в `%LocalAppData%\AutoRAW\user files\Profile`). Для **Кроссовок** референсы и каталог **Zona** лежат в **`reference\Sneakers`** и **`zona\Sneakers`**. У других профилей — своя подпапка с тем же именем, что и профиль (`reference\<имя>\`, `zona\<имя>\`). Свои каталоги reference/zona задаются при **добавлении пользовательского профиля** (структура папки профиля). Сопоставление кадров по режимам «Стандартный / Руслан / Маша» в коде сохранено; пункт **«Фотограф»** в меню временно скрыт.
2. **Папка «Товар»** — каталог с исходными снимками (включая **все вложенные подпапки**). **Стандартный:** в каждой папке файлы по порядку имени — 1-й→`01`, 2-й→`02`, … 8-й→`08`. **Руслан:** номер из имени (`1.NEF`, `тип3`…). **Маша:** по совпадению имени с референсом. Каталоги `webp` и `jpg` внутри входа не обрабатываются повторно. Папку можно выбрать через **Обзор…** или **перетащить** в поле «Товар (вход)».
3. **Папка выхода** — необязательна.
   - **Пустая** — создаётся `webp` или `jpg` **в корне «Товар»**, внутри — те же подпапки, что во входе: `Товар\webp\партия_A\01.webp`.
   - **Задана** — в неё копируется та же структура подпапок: `D:\готово\партия_A\01.webp`.
4. Кнопка **«Запустить кадрирование»** во время работы становится **«Пауза»** / **«Продолжить»**; узкая **«Отмена»** справа прерывает пакет.
5. Галочка **«Сохранять результат в WebP»** (по умолчанию включена) — состояние в `%AppData%\AutoRAW\export_prefs.json`.
6. Над журналом — **полоса прогресса** (`94/100`, при паузе — «На паузе», по завершении — «Готово 100/100»).
7. **Журнал**: **⧉** — отдельное окно без системной шапки (перетаскивание за заголовок «Журнал», изменение высоты); повторное **⧉** — снова внизу главного окна. **✕** — скрыть. Цвета строк: ошибки и отмена — красный, пауза — синий, итог «Готово!» — зелёный.

### Превью и ползунок «Анализ»

- Пока пересчитывается превью (**Референс · До · После**), над блоком показываются **текст статуса** и **полоска прогресса**, а внутри каждой ячейки превью — **круглый индикатор загрузки**, пока картинка не готова. Блок превью можно скрыть: **Вид → Окна → Показать превью**.
- В группе **Превью** есть ползунок **«Анализ (длинная сторона)»** — он влияет на расчёт кропа по референсу. При движении ползунка превью снова пересчитывается **с тем же индикатором загрузки**. После правок в **Редакторе положения кадра** колонка **После** на главном экране обновляется вместе с сохранением правок в json (в том числе при перетаскивании результата мышью — без паузы на экране редактора).
- **Вид → Редактор положения кадра…** — отдельное окно: список **подпапок** папки «Товар» (**несколько папок**: Ctrl как в проводнике), миниатюры файлов в пропорции выходного кадра **1400×1050** с превью **как после кропа и правок** (кэшируются и обновляются после сохранения правок). Над списком файлов: **«Все файлы» / «Фильтр по имени»**, опционально **«Группировать по имени файла»**. Имя файла — **двойной щелчок** для переименования на диске (json пропусков и правок подстраиваются). **Галочка «в очередь»** у каждого файла — см. `%AppData%\AutoRAW\editor_skipped_files.json`. По превью миниатюры — клик **открывает** режим правки (без Ctrl; при Ctrl/Shift выделение строк как обычно). У списка папок — **«Пропустить»** / **«Вернуть»** (`editor_ignored_folders.json`). В режиме правки большие превью «Референс» и «Результат» показывают **круглый индикатор загрузки** на время расчёта. Меню **Файл → Применить профиль**: только эта фотография (json), ко всем файлам с **тем же именем** в текущем списке браузера, **Сохранить как профиль…**; отдельно **Файл → Сохранить для профиля (текущий кадр)** — умолчание профиля по номеру кадра в `manual_shot_adjust.json`. Справа кнопки **«Применить изменения»** (явное сохранение только для этого файла) и **«Сбросить»** (убрать per-file переопределение и подставить настройки профиля по кадру). Текст подсказки внизу окна на несколько секунд подтверждает сохранение и другие действия. Превью результата в режиме правки считается **в облегчённом разрешении** во время движения ползунков и перетаскивания; через паузу — более детальный кадр (только экран). В режиме правки вы видите **превью будущего файла** в **пикселях референса**: под этот прямоугольник подгоняется полный исходник (**cover**), дальше задаются **смещение X/Y**, **масштаб %**, **поворот °**. **Сетка выравнивания** из папки `zona` — только на экране. При закрытии окна редактор по-прежнему **автоматически** записывает текущие ползунки для открытого файла в per-file слот.
- После сохранения на **превью** и при **пакетном экспорте** используется одна и та же геометрия: **полный** кадр исходника → ручные **смещение / масштаб / поворот** → **выход** строго в размере выбранного **референса**; **сетка** на диск не попадает.

### Как выбирается кадр

Все входные файлы обрабатываются **одинаково**: размер и пропорции результата задаёт файл из папки `reference` для выбранного номера кадра (01…08); положение и масштаб товара — **ручные** настройки редактора / json. Папка `zona` опциональна (**PNG сетки** для редактора).

Если профиль перешёл в состояние черновика **«Несохранённые изменения»** (например после смены путей или цветокоррекции относительно исходного пакета), сохраните его через меню **Профиль**: **Применить черновик к исходному профилю…** или **Сохранить черновик как новый профиль…**.

## Свой профиль

**Профиль → Добавить новый профиль…** — укажите **папку профиля** (готовый bundle с `profile.json`, каталог из `profiles\<slug>` проекта, или папки `reference` / `zona`) и **имя** (можно оставить предложенное из имени папки). Профиль **копируется** в `%LocalAppData%\AutoRAW\user files\Profile\<имя>\` (`reference\<имя>\`, `zona\<имя>\`, `setting`, `profile.json`).

**Профиль → Перейти в мой каталог профилей…** — открывает `%LocalAppData%\AutoRAW\user files\Profile` в Проводнике (можно вручную копировать или править папки профилей).

## Профили в комплекте программы (сборка из репозитория)

Чтобы пресет попал **ко всем пользователям** в меню «Профиль» вместе с установщиком, в корне репозитория заведите:

1. **`reference\<slug>\`** и **`zona\<slug>\`** — те же данные, что для пользовательского профиля (`slug` — короткое имя папки на латинице, без пробелов).
2. **`profiles\<slug>\profile.json`** — манифест с полем **`displayName`** (имя в меню, можно по-русски) и **`color`** (как в `profile.json` у профиля в `%LocalAppData%`).

При `dotnet build` / `dotnet publish` содержимое **`profiles\`** копируется рядом с `AutoRAW.exe`. Профили из этого каталога появляются в меню **Профиль → Стандартные**. Профиль «Кроссовки» задаётся через **`profiles\Sneakers\`** и папки **`reference\Sneakers`**, **`zona\Sneakers`**.

## Уведомления в Telegram

**Настройки → Telegram ZONA…** — при **успешном завершении пакета**, **ошибке** и **отмене** ZONA может отправить сообщение в ваш чат (токен бота от @BotFather и ID чата). Настройки сохраняются в `%AppData%\AutoRAW\telegram_zona.json`. После сохранения токен и chat id можно сменить только через **«Изменить…»** и пароль.

Сообщение о **готовом пакете** выглядит так: сначала короткая фраза от ZONA, затем строка **✅ Успешно: N, ошибок: M, всего: K. Время: …**, затем **Профиль: имя** — без отдельной служебной первой строки «пакет готов».

### Формат файлов после кадрирования

| Папка выхода | Галочка WebP | Куда и как сохраняется |
|--------------|----------------|------------------------|
| **Не задана** | Включена (по умолчанию) | `<Папка товара>\webp\<подпапка>\*.webp` |
| **Не задана** | Снята | `<Папка товара>\jpg\<подпапка>\*.jpg` |
| **Задана** | Включена | `<Папка выхода>\<подпапка>\*.webp` |
| **Задана** | Снята | `<Папка выхода>\<подпапка>\*.jpg` |

Пример: вход `D:\фото\партия\01.nef`, выход пустой, WebP → `D:\фото\webp\партия\01.webp`.

Готовые кадры по профилю «Кроссовки» приводятся к размеру **референса** (например **1400×1050** px).
